## JavaScript and TypeScript overlay

- Match the project's JavaScript or TypeScript choice, module system, runtime targets, and compiler strictness; do not force a TypeScript migration.
- Use modern syntax, `const`/`let`, strict equality, and small modules within the configured target.
- Respect browser, Node.js, edge, worker, or framework runtime differences.
- Handle promises and async failures explicitly, including cleanup or cancellation when relevant.
- Follow existing React or other framework conventions for components, state, rendering, and accessibility.
- Use the established Jest, Vitest, Testing Library, native runner, and Vite/Webpack/esbuild workflow as applicable.
